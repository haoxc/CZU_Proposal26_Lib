# 课题多维度评估

## 一、与申报通知的匹配度评估

### 结论

匹配度高，但表述方式必须控制。

### 原因

- 与通知中的 `AI 驱动的未来学习中心智慧学习范式构建研究` 高度吻合
- 与 `智慧图书馆环境下学习支持服务的转型与优化` 高度吻合
- 与 `高校图书馆支撑“新质生产力”人才培养的知识服务创新研究` 也有较强对应关系

### 风险

如果标题和摘要过度强调机器人、插座、硬件联动，评审容易把它看成工程项目，而不是教改研究项目。

## 二、创新性评估

### 结论

创新性强，公开可见同类一体化案例较少。

### 亮点

- 以学生个人智能体作为统一入口
- 将图书馆服务、学习档案和空间联动放在同一服务链中
- 强调未来学习中心场景下的学习支持服务模式创新

### 建议

创新性要通过“模式设计 + 实施路径 + 评价机制”来表达，不要只通过功能堆叠来表达。

## 三、可行性评估

### 软件部分

可行性高。

- 企业微信入口、知识库、AI 问答、借阅查询、学习档案基础版，均属于现阶段成熟可落地能力。
- 推荐采用 `Dify + Python/FastAPI + PostgreSQL + Redis` 的组合，先快后稳。

相关参考：

- Dify 可通过自托管 Docker Compose 快速部署，官方文档给出了最低硬件与部署要求。
  来源：[Dify Docs](https://docs.dify.ai/getting-started/install-self-hosted/docker-compose)
- Dify 已提供 `Agent`、`Tool Node`、知识库 API 等能力，适合低门槛构建带工具调用的智能体应用。
  来源：[Agent](https://docs.dify.ai/en/guides/workflow/node/agent)
  来源：[Tool Node](https://docs.dify.ai/en/use-dify/nodes/tools)
  来源：[Manage Knowledge via API](https://docs.dify.ai/en/use-dify/knowledge/manage-knowledge/maintain-dataset-via-api)

### 多智能体深度编排部分

可行，但不建议作为一期主路线。

- LangGraph 更适合复杂状态编排和较强工程控制，但企业级自托管与观测能力更偏进阶，对团队要求更高。
  来源：[LangGraph Self-hosted / LangSmith](https://docs.langchain.com/langgraph-platform/self-hosted)
- AutoGen、CrewAI 更适合复杂多角色协作任务，但对图书馆一期服务闭环来说，收益未必高于复杂度。
  来源：[AutoGen](https://autogenhub.github.io/autogen/docs/Use-Cases/agent_chat/)
  来源：[CrewAI Docs](https://docs.crewai.com/en/introduction)

### 综合判断

- 一期：`Dify 为主，Python 服务托底`
- 二期：在复杂流程或评测环节引入 `LangGraph`

## 四、硬件可行性与成本评估

### 结论

技术可行，工程复杂度中等偏高，适合小范围试点，不适合申报期承诺全馆铺开。

### 技术路径

- 每个座位设置固定二维码，二维码本身成本极低
- 学生扫码后由后端完成 `身份识别 -> 座位签到 -> 设备开关指令`
- 设备层通过智能继电器、智能插座或受控 PDU 实现台灯与插座上电
- 后台记录状态，离座、超时、异常时自动断电

### 成本判断

基于 `2026-04-23` 可检索到的公开商品和方案样本，单纯元件价格不高，真正成本主要在布线、配电安全、安装和运维：

- 智能继电器/开关模块：公开零售样本约在 `10-40 美元` 或相近人民币区间
  参考：[Gearbest 样本 1](https://www.gearbest.ma/zh-CN/product/tuya-zigbee-wifi-smart-home-relay-module-garage-door-relay-ac-dc-7-32v-85-250v-works-alexa-google-assistant-smart-light-switch/)
  参考：[Gearbest 样本 2](https://www.gearbest.ma/zh-CN/product/tuya-zigbee-3-0-smart-light-switch-module-diy-home-automation-breaker-relay-with-2-way-control-compatible-with-alexa-google/?rating=3)
- 智能 PDU 市场公开列表显示，企业级远程管理型设备通常显著高于普通插座
  参考：[PDU 市场列表](https://www.chazuo.com/pdu/filter/24/)
- 图书馆座位预约联动灯光的行业方案已经存在，说明场景本身可实现
  参考：[电子纸图书馆座位预约系统](https://www.einkcn.com/epd/?id=41)

### 工程估算

以下为基于公开元件和常见施工逻辑的工程推断，不是采购报价：

- `20-50` 个座位试点：通常可控制在 `2 万 - 8 万元`
- 全馆改造：取决于桌体结构、布线、回路改造和安全验收，通常会远高于元件成本

### 建议

- 先做 `20-50` 个座位试点
- 优先用低压控制、集中供电和标准电工施工
- 必须把消防、电气安全、运维责任写进试点方案

## 五、数据治理与伦理评估

### 结论

这是课题能否顺利评审和落地的关键风险点。

### 主要风险

- 学习档案容易被误解为监控或评价
- 榜单机制可能引发标签化
- 多系统数据打通涉及最小必要原则和权限边界

### 建议

- 学习档案默认面向学生本人和服务改进，不做刚性考核
- 榜单优先采用 `自愿参与、匿名展示、分组展示`
- 明确数据来源、用途、保存期限和权限控制

## 六、申报竞争力评估

### 优势

- 选题符合省内当前导向
- 场景鲜明，学校特色和实践想象空间较强
- 能形成“研究 + 建设 + 成果”三位一体叙事

### 短板

- 若前期基础证明不足，容易被质疑“概念过大”
- 若缺少经费或跨部门协同承诺，容易影响可信度

### 提升办法

- 准备馆内基础、过往智慧服务、信息化协同证据
- 强化一期试点边界，减少不确定性
- 争取经费、后勤、信息化的明确支持口径

## 七、最终判断

这个题目值得报，且有机会做出辨识度。但最稳的打法不是“把所有功能都写满”，而是把项目收敛成：

`一个主场景 + 一条实施路径 + 一套评价机制 + 一个空间试点`

这样既保住教改研究属性，也保住后续实际落地的概率。
