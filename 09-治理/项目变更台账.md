# 项目变更台账

适用项目：

- 江苏省高校图工委教改课题申报与落地一体化项目

## 作用

这份台账用于解决两类问题：

- 后续审计时，能追溯“为什么改、改了什么、影响了什么”
- 推进过程中，能识别项目是否已经出现目标漂移、范围膨胀或口径失真

它不替代日常任务管理，也不替代正文文档。它只负责记录“值得被回溯的变化”。

## 何时必须登记

以下变更，原则上都要登记：

- 课题标题变更
- 主叙事或申报口径变更
- 一期/二期边界调整
- 重点课题 / 一般课题选择变化
- 目录结构调整
- 核心技术路线变化
- 关键成果承诺变化
- 配套经费口径变化
- 关键风险判断变化
- 影响送审节奏的重大调整

## 目标漂移预警规则

出现以下情况时，应视为“目标漂移预警”：

- 新增内容没有服务总目标，却持续占用时间
- 工程实现不断膨胀，教改主线被削弱
- 文稿越来越多，但关键硬信息仍未补齐
- 方案反复改题，却没有显著提升推荐概率
- 评审风险未清零，却开始过早展开实施细节

若出现预警，先回到 `10-项目工作` 看当前优先级，再决定是否接受变更。

## 台账字段说明

| 字段 | 含义 |
| --- | --- |
| 变更编号 | 唯一编号，建议使用 `C001` 递增 |
| 日期 | 变更确认日期 |
| 变更对象 | 被调整的对象，如标题、目录、技术路线、MOC |
| 变更类型 | 新增 / 调整 / 合并 / 删除 / 重构 |
| 变更内容 | 具体改了什么 |
| 触发原因 | 为什么要改 |
| 影响范围 | 会影响哪些文档、判断或后续动作 |
| 对总目标影响 | 增强 / 中性 / 有漂移风险 |
| 是否需要回溯同步 | 是否需要同步改其他文档 |
| 当前状态 | 已确认 / 已执行 / 待验证 / 已废止 |

## 后续使用规则

- 每次重大调整，先记一条变更，再改正文。
- 如果一项变更会影响三个以上文档，必须在这里先写清影响范围。
- 如果一项变更可能导致目标漂移，必须在“对总目标影响”中明确标为 `有漂移风险`。
- 若某项变更后来被否定，不删除原记录，改状态为 `已废止`，并补一条新的反向变更。
