# GitHub 仓库初始化 skill 决策

时间：`2026-04-24`

## 决策内容

新增一个窄职责 skill：

`git-github-bootstrap`

## 适用范围

该 skill 负责以下场景：

- 本地目录尚未初始化为 Git 仓库
- 需要绑定或校正 GitHub 仓库地址
- 需要补最小 `.gitignore`
- 需要为首次推送接通 `gh` 凭据
- 需要形成首个基线提交

## 边界

该 skill 不负责长期的 GitHub 提交流程治理，也不负责复杂 push 故障诊断。

如果问题已经进入：

- SSH / HTTPS 路径异常
- 认证失败反复出现
- 远端传输异常反复出现

则应切换到现有的 `github-push` 路径处理。

## 管理含义

这样做的目的，是把“仓库初始化与绑定”从“推送故障诊断”中分离出来，避免一个 skill 同时承担两类职责，导致触发范围过宽、动作边界模糊。
