# Directory Model

## Purpose

This reference defines a proposal-management directory model derived from the current project’s working standard.

## Top-Level Structure

```text
00-总览/
01-规范/
02-管理/
09-治理/
10-项目工作/
60-资产/
90-存档/
98-附件/
99-设置/
```

## Roles

| Directory | Role |
| --- | --- |
| `00-总览` | Global entry, stable navigation, key links |
| `01-规范` | Current rules: directory, naming, writing style |
| `02-管理` | Plans, milestones, risks, rhythm, coordination |
| `09-治理` | Audit results, decisions, change records, rule adoption |
| `10-项目工作` | Single active work area for proposal and related work |
| `60-资产` | Reusable assets, templates, skills, visuals |
| `90-存档` | Archived material that left the active line |
| `98-附件` | Raw attachments, source PDFs, exports |
| `99-设置` | Templates, scripts, Obsidian settings, helper standards |

## Active Work Modules

Inside `10-项目工作`, use:

```text
10-课题申报/
20-方案设计/
30-落地实施/
40-调研案例/
50-评估论证/
```

## Why This Model

This model solves two common proposal-repo problems at the same time:

- active work needs a single clear entry
- different proposal functions still need boundaries

If you flatten these modules into many top-level work folders, the workspace feels scattered.

If you collapse everything into one directory with no modules, boundary drift grows quickly.
