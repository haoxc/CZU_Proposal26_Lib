# Naming And Lifecycle

## Core Rule

Proposal workspaces need different naming behavior for different lifecycle states.

## Stable Baseline Files

Usually do **not** force dates on:

- current rules
- ledgers
- current accepted baselines
- module MOCs
- folder notes

Examples:

- `项目目录规范.md`
- `项目变更台账.md`
- `10-项目工作.md`
- `10-课题申报.md`

## One-Time Process Files

Should include `yy.MMdd`.

Examples:

- `项目管理目录审计报告-26.0423.md`
- `工作目录重构说明-26.0423.md`
- `当前申请书专家审计意见-26.0423.md`
- `高校图书馆AI创新实践调研-26.0423.md`

Typical process files:

- audit reports
- review snapshots
- one-time analyses
- restructuring notes
- naming decisions
- excerpt notes

## Draft And Lifecycle Exposure

Draft-like files should expose lifecycle state:

- `草案`
- `填报稿`
- `审计意见`
- `要点摘录`

Avoid vague names like:

- `方案.md`
- `材料.md`
- `文稿最终版.md`

## Folder Note Exception

If the workspace uses Obsidian `folder-notes`, folder-note files may match the folder name exactly and keep module numbering.

Examples:

- `10-项目工作.md`
- `10-课题申报.md`
- `50-评估论证.md`

These are exceptions by design, not naming violations.
